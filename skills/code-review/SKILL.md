# 代码审查 Skill

这个 Skill 面向中文用户，用于审查一个本地项目或仓库，输出结构清晰、证据充分、可执行的项目审查结果。

## 工作流程

1. 先用 `workspace_files` 了解项目目录、配置文件、测试文件和核心源码分布。
2. 按需用 `file_reader` 阅读 README、配置、入口文件、核心模块和关键测试。
3. 总结项目的技术栈、功能边界、运行链路、主要模块和数据流。
4. 识别风险时要绑定具体文件或现象，区分“确定问题”和“需要进一步确认”。
5. 检查测试覆盖：已有测试覆盖什么、缺哪些关键路径、哪些验证命令值得运行。
6. 用户要求保存时，用 `markdown_writer` 写出中文审查报告。

## 输出要求

- 默认用中文回答。
- 先给结论，再给依据和建议。
- 不要编造文件、测试结果或项目事实；没读到就说明没读到。
- 如果信息不足，优先请求工具读取，而不是凭印象下判断。
- 建议要可执行，例如指出应补的测试、应拆的模块或应验证的命令。

## 推荐结构

```markdown

```
