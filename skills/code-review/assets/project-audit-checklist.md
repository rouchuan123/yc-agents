# 项目体检快速清单

用于本地仓库审查。先建立证据，再下结论；未读取或未运行的内容必须标注为未确认。

## 体检前

- [ ] 已读取 README、项目说明或入口文档。
- [ ] 已识别语言、框架、包管理器、运行入口和测试命令。
- [ ] 已列出配置文件、核心源码目录、测试目录、脚本目录和输出目录。
- [ ] 已确认是否需要按技术栈读取 `references/` 中的语言或框架指南。

## 架构与边界

- [ ] 入口、编排层、业务逻辑、工具/服务、数据访问或状态管理边界清楚。
- [ ] 没有明显循环依赖、跨层调用、上帝对象或大泥球结构。
- [ ] 配置、密钥、运行状态和业务逻辑没有混在一起。
- [ ] 新增或核心模块位置符合项目已有组织方式。

## 关键链路

- [ ] 至少追踪一条主要 CLI/API/任务链路。
- [ ] 链路覆盖入口、参数解析、核心逻辑、外部调用、错误处理和结果输出。
- [ ] 链路能对应到现有测试或明确记录测试缺口。

## 正确性与质量

- [ ] 边界条件、空输入、异常路径、并发或重入风险已检查。
- [ ] 没有重复造轮子、参数膨胀、抽象泄漏、magic string、深层嵌套或复制粘贴变体。
- [ ] 状态更新、文件操作、数据库写入或外部调用有必要的幂等和失败处理。
- [ ] 代码命名、模块职责和注释服务于理解，而不是掩盖复杂度。

## 安全

- [ ] 没有硬编码密钥、令牌、密码或敏感 URL。
- [ ] 外部输入经过校验、规范化或安全解析。
- [ ] 权限检查和资源访问边界明确。
- [ ] 日志、错误消息和输出不会泄露敏感数据。

## 性能与可靠性

- [ ] 没有明显的 N+1 查询、全量读取再过滤、无界循环或无界并发。
- [ ] 大文件、大列表、网络调用和外部服务有超时、分页、限流或缓存策略。
- [ ] 失败路径有清晰恢复、重试或降级策略。
- [ ] 长耗时任务不会阻塞关键交互路径。

## 测试缺口

- [ ] 已确认现有测试覆盖哪些入口和关键行为。
- [ ] 已指出缺少的边界条件、错误路径、权限边界、回归风险和集成路径。
- [ ] 已建议可执行的验证命令。
- [ ] 没有把未运行的测试描述成已通过。

## 输出前

- [ ] 每个主要发现都有文件路径证据，能给行号时给行号。
- [ ] 每个发现都有影响、触发条件和修复方向。
- [ ] 已区分已确认事实、基于代码的推断、未确认事项。
- [ ] 建议下一步可执行，避免泛泛而谈。
