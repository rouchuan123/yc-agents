# 模板规则结构

这份文件给后续修改者看的规则分组说明，用来理解 `document-format-normalizer` 如何把模板要求翻译成可执行的 DOCX 格式规则。这里描述的是“规则字段怎么改”，不是用户侧操作文档。

## 总体原则

- 规则键名保持英文，例如 `page`、`styles.body`、`table_of_contents`。这些键名会被 Python 代码读取，不要翻译。
- 说明文字、注释和模板解释优先使用中文，方便中国用户和维护者理解。
- 长度单位优先写清楚：厘米用 `cm`，字号用 `pt`。
- 不确定 Word 对象能否稳定重建时，规则里不要硬承诺，交给审计报告提示人工复核。

## 规则分组

| 分组 | 含义 | 常见修改点 |
| --- | --- | --- |
| `page` | 页面设置 | 纸张、方向、页边距、页眉页脚距离 |
| `styles.body` | 正文样式 | 中文字体、英文字体、字号、行距、首行缩进、段前段后、对齐方式 |
| `styles.heading_1` | 一级标题 | 字体、字号、加粗、居中、目录级别、是否另起一页 |
| `styles.heading_2` | 二级标题 | 字体、字号、加粗、左对齐、目录级别 |
| `styles.heading_3` | 三级标题 | 字体、字号、加粗、左对齐、目录级别 |
| `styles.caption` | 图题和表题 | 字体、字号、居中、段前段后 |
| `table_of_contents` | 目录字段 | 是否插入目录、包含哪些标题级别 |
| `page_numbers` | 页码字段 | 是否插入页码、页码位置、起始编号 |
| `tables` | 表格预设 | 列宽、行高、单元格边距、表格对齐方式 |

## 内置模板

当前内置模板以 `report-standard` 为权威版本。它适合报告、项目材料、制度文件等常见中文文档的初稿格式整理。

如果用户上传 `.docx` 模板，工具会尽量读取模板中的页面边距和 `Normal` 样式。无法稳定读取的复杂样式不会强行猜测，应在审计报告中提示用户人工复核。

## 修改建议

- 想改页边距：优先修改 `page`。
- 想改正文：优先修改 `styles.body`，不要只改某几个段落。
- 想改标题目录层级：同时检查 `styles.heading_*` 和 `table_of_contents`。
- 想改页码：检查 `page_numbers`，并用生成后的 `.audit.md` 复核。
- 想新增模板：先复制 `report-standard` 的结构，再逐项改动，避免遗漏审计逻辑依赖的字段。
