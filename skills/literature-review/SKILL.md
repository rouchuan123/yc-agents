# 文献综述 Skill

## 什么时候使用

当用户需要整理文献、总结论文、比较多篇研究、撰写国内外研究现状、提炼研究趋势、分析研究不足或为开题报告准备文献依据时，使用这个 Skill。

这个 Skill 可以服务不同专业方向。通信工程、工业遮挡失联、定位、通信状态感知、Spring Boot 等内容只能作为用户背景、资料内容或检索关键词，不能写死进核心 Runtime。

## 输入信息检查

开始前先判断用户是否已经提供以下信息：

- 文献文件路径、资料目录或 RAG 知识库内容
- 用户希望综述的主题或关键词
- 需要综述的范围，例如国内研究、国外研究、近五年研究或某一技术路线
- 输出用途，例如开题报告、论文正文、汇报材料或个人学习笔记
- 是否需要引用格式、表格对比或 Markdown 文件
- 用户已有观点、导师要求或学校模板

如果用户没有提供任何文献或资料，不要编造论文。可以先给出文献综述框架和资料收集清单。

## 工作流程

1. 明确用户要做的是文献摘要、论文对比、研究现状、研究趋势、研究不足，还是完整文献综述草稿。
2. 检查用户是否提供了文献资料、Word 文档路径、资料目录或检索关键词。
3. 如果用户询问当前工作区有什么文件，可以请求使用 `workspace_files` 列出可读文件。
4. 如果用户提供或选定 `.docx`、`.pdf`、`.md` 或 `.txt` 文件，可以请求使用 `file_reader` 读取正文。
5. 如果需要从已有资料库中查找依据，可以请求使用 `rag_search` 检索相关片段。
6. 提取每篇文献的研究问题、方法、实验或数据、结论、不足和可借鉴点。
7. 按主题、方法、时间线或问题链组织综述，不要简单堆叠每篇论文摘要。
8. 输出时明确哪些内容来自用户资料，哪些只是待补充方向。
9. 如果用户明确要求保存草稿，可以请求使用 `markdown_writer` 写出 Markdown 文件。
10. 最后给出下一步建议，例如还缺哪些文献、应该补哪类关键词、哪些部分需要导师确认。

## 文献提取维度

整理单篇文献时，优先提取：

- 研究对象或问题
- 使用的方法、模型、系统或算法
- 数据来源、实验场景或验证方式
- 主要结论
- 贡献或优点
- 局限或不足
- 可以借鉴到用户课题的内容
- 和用户研究方向的关系

不要只摘抄摘要，要尽量转化成用户能用于开题或论文写作的材料。

## 综述组织方式

根据用户需求选择合适结构：

- 按主题组织：适合多个研究方向并列对比
- 按方法组织：适合比较不同算法、系统或技术路线
- 按时间线组织：适合描述某个领域的发展过程
- 按问题链组织：适合围绕“现有问题 -> 解决方案 -> 不足 -> 本文切入点”展开
- 按国内外研究现状组织：适合开题报告格式

如果用户没有指定结构，优先使用“主题 + 问题链”的方式，因为它更容易自然引出用户自己的研究切入点。

## 输出类型

常见输出包括：

- 单篇文献摘要
- 多篇文献对比表
- 国内外研究现状段落
- 研究趋势总结
- 研究不足分析
- 可借鉴方法清单
- 文献综述草稿
- 下一步检索关键词和资料清单

## 工具使用原则

- 用户询问工作区文件、资料目录或“你能读取到什么”时，可以请求 `workspace_files` 列出当前工作区可读文件。
- 用户提供或选择 `.docx`、`.pdf`、`.md`、`.txt` 文件时，可以请求 `file_reader` 读取正文内容。
- `docx_reader` 保留兼容旧流程；新文档读取优先使用 `file_reader`。
- 只有需要从已有资料、笔记或知识库中查找依据时，才请求 `rag_search`。
- 只有用户明确要求保存、导出或生成文件时，才请求 `markdown_writer`。
- 没有资料时不要编造作者、年份、题名、期刊、实验数据或引用。
- 如果检索结果不足，要明确说明“当前资料不足”，并给出补充资料建议。

## 约束

- 不编造参考文献。
- 不把没有检索到的内容说成已有研究结论。
- 不把文献综述写成资料堆砌。
- 不默认所有研究都和用户课题强相关。
- 不主动写文件，除非用户明确要求。
- 默认使用 Markdown 输出，除非用户要求其他格式。
- 不执行 `scripts/` 下的脚本；脚本执行必须等后续 ToolGateway、权限检查、用户确认和 Trace 支持。

## 完成检查项

完成任务前检查：

- 已明确用户要做的是摘要、对比、研究现状、趋势分析还是完整综述。
- 已说明当前回答基于哪些资料或检索依据。
- 如果资料不足，已明确指出缺口。
- 如果涉及多篇文献，已按主题、方法、时间线或问题链组织，而不是简单堆叠。
- 已提取研究问题、方法、结论、不足和可借鉴点。
- 没有编造作者、年份、题名、期刊或实验数据。
- 如果用户要求保存文件，已准备调用 `markdown_writer`。
- 已给出下一步检索关键词或资料补充建议。
