# 开题报告 Skill

## 什么时候使用

当用户需要准备研究生开题报告、论文选题、研究计划、研究背景、技术路线、可行性分析、风险分析或下一步资料清单时，使用这个 Skill。

## 工作流程

1. 先理解用户的研究方向、学校要求、导师反馈、论文类型和当前约束。
2. 判断用户是否已经提供文档、笔记、模板、导师意见或已有草稿。
3. 如果需要事实依据，先读取资料或检索相关片段，再组织观点。
4. 帮用户梳理候选题目、研究背景、研究问题、技术路线、可行性、风险和下一步计划。
5. 如果用户要求生成文件，再调用写入工具保存 Markdown 草稿。
6. 在输出最终内容前，尽量说明当前版本的假设和还缺哪些材料。

## 约束

- 不编造参考文献、实验结果或导师意见。
- 用户没有要求写文件时，不主动生成文件。
- 默认使用 Markdown 输出，除非用户要求其他格式。
- 不把某个具体专业、具体论文题目或具体工程场景写死进 Runtime。
- 通信工程、工业遮挡失联、Spring Boot 等内容只作为用户背景或 Skill 知识使用，不写进核心框架逻辑。

## 完成检查项

- 已明确用户的研究方向或当前问题。
- 已给出候选题目、报告结构或写作建议。
- 需要研究背景时，已补充背景逻辑。
- 已说明研究目标、研究问题或核心任务。
- 已描述技术路线或系统方案思路。
- 已列出可行性、风险或待确认问题。
- 已给出下一步行动建议或资料清单。
